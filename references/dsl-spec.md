# DSL 구문 사양

UP 규칙 작성·수정시 유일하게 허용되는 문법. 자연어 산문 ✗.

---

## 구문 목록

| 구문 | 형식 | 용도 |
|------|------|------|
| 정의 | `NAME ::= "형식" @ 적용조건 \| 위반시결과` | 규칙 선언 |
| 열거 | `ENUM {값1:의미, 값2:의미}` | 허용값 목록 |
| 조건분기 | `IF condition → action \| ELSE → fallback` | 분기 로직 |
| 예외 | `EXCEPT context: 예외규칙` | 조건부 면제 |
| 순차 | `PIPE: ①단계 → ②단계 → ③단계` | 실행 순서 |
| 게이트 | `GATE: conditions ALL → 통과 \| IF 미충족 → 차단` | 진행 조건 |
| 참조 | `@ref §섹션.규칙명` | 다른 규칙 참조 |
| 범위 | `SCOPE / LIMIT / BOUNDARY` | 적용 범위 한정 |
| 금지/필수 | `✗ "금지항목"` · `RULE: 필수항목` · `FAIL = 위반` | 제약 조건 |
| 면제 | `SKIP: 면제조건` | 규칙 적용 제외 |
| 매핑 | `MAP: "원본" → "치환대상"` | 변환 테이블 |
| 후처리 | `POST: 완료 후 필수 행동` | 사후 조건 |
| 예시 | `EXAMPLE: 내용` | 추론불가 사항만 최소한 |

---

## 작성 규칙

1. **들여쓰기** = 스페이스 2칸 (탭 ✗)
2. **기호 연산자**: → ✗ ≥ ≤ × ∝ 적극 활용
3. **DSL 키워드**: IF, ENUM, GATE, PIPE, RULE, FAIL, SKIP, POST, MAP, SCOPE, LIMIT, EXCEPT — 대문자 유지
4. **1규칙 1블록**: 관련 하위규칙은 들여쓰기로 종속
5. **주석 불가**: DSL에 주석 구문 없음. 이유 설명은 `REASON:` 또는 `DESIGN:` 키워드 사용

## 안티패턴

| ✗ 금지 | ✓ 올바른 형태 |
|--------|--------------|
| "확신도는 70 이상일 때 검증이 필요합니다" | `IF ≥70 → 검증행위 필수` |
| "가능하면 Python을 사용해주세요" | `IF 연산자≥2 → Python필수` |
| "참고로 이 규칙은..." | `REASON: ...` 또는 `DESIGN: ...` |
